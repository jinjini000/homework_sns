### 🌟 My-SNS 프론트엔드 구현 과제

#### **프로젝트 소개**

이 프로젝트는 미리 구축된 **Node.js 백엔드 서버**를 활용하여 **프론트엔드(Client-Side)** 구현을 학습하는 데 중점을 둔 과제 결과물입니다. 사용자 인증, 게시물 조회 및 관리 등 SNS의 주요 기능을 클라이언트 측에서 구현하고, 백엔드 API와 성공적으로 통신하는 것에 초점을 맞췄습니다.

> **💡 참고:** 이 프로젝트는 기능 구현에 집중했으므로, **예외 처리** 로직은 포함되어 있지 않습니다.

-----

### ⚙️ 기술 스택 (Tech Stack)

| 구분 | 기술 스택 | 사용 목적 |
| :--- | :--- | :--- |
| **프론트엔드** | **HTML5, CSS3, JavaScript (Vanilla JS)** | 사용자 인터페이스(UI) 구성 및 동적 기능 구현 |
| **백엔드 (사전 구축)** | **Node.js, Express** | 서버 런타임 및 API 제공 |
| **데이터베이스** | **MongoDB** | 사용자 정보 및 게시물 데이터 저장 |

-----

### 🚀 시작 방법 (Getting Started)

#### 1\. 의존성 설치

프로젝트에 필요한 Node.js 모듈들을 설치합니다.

```bash
npm i
```

#### 2\. 서버 실행

개발 환경에서 서버를 실행합니다.

```bash
npm run dev
```

#### 3\. 접속

서버가 실행되면 다음 주소로 접속하여 메인 화면을 확인할 수 있습니다.

```
http://127.0.0.1/
```

### 🔑 주요 구현 기능

이 프로젝트에서 프론트엔드가 백엔드와 연동하여 구현한 핵심 기능들입니다.

#### **1. 사용자 인증 (로그인/회원가입)**

| 기능 | 프론트엔드 역할 | 백엔드 역할 |
| :--- | :--- | :--- |
| **로그인** | 사용자 입력 정보(ID/PW)를 API로 전송 | MongoDB의 사용자 정보와 비교 후 **JWT 토큰**을 생성하여 **HTTP 쿠키**로 전송 |
| **회원가입** | 사용자 정보 입력 및 API로 전송 | Mongoose를 이용해 MongoDB에 저장하며, **bcrypt**를 이용해 비밀번호를 **암호화**하여 저장 |

#### **2. 포스트 관리 (게시물)**

| 기능 | 설명 |
| :--- | :--- |
| **전체 게시물 조회** | 메인 화면에서 전체 게시물 목록을 API 요청을 통해 불러와 렌더링 |
| **특정 사용자 게시물 조회** | 사용자 ID를 기준으로 필터링된 게시물 목록을 API로 요청하여 표시 |
| **상세 게시물 조회** | 게시물 클릭 시 해당 게시물의 고유 ID를 이용해 상세 정보를 API로 요청 및 화면에 표시 |
| **게시물 수정/삭제** | 해당 게시물을 **작성한 사용자**만 수정/삭제 버튼을 볼 수 있으며, 클릭 시 API를 통해 데이터 수정/삭제 요청 |
